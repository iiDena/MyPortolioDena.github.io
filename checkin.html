Airport Checkin

Project Case

In this case, you have to design a new interface for check-in stands for Copenhagen airport. This check-in stand must have a number of new features which help to provide users the good user experience based on the needs they now have. You must have 3 features with you your design in addition to a normal check-in process and as a minimum you must come up with one yourself. Each group will be assigned an airline and a target group and then it's up to you to analyze their needs when they stand in front of a check-in stand. It is important that you consider the airline's business in your design the users.

Details

  • Topic: User Experience - User Journey
  • Softwares/tools: Figma
  • Project date: August - 2021
  • Project Website: Figma Project

Project requirements

  • Users: Wheelchair users
  • Airline: Norwegian
  • Needs: Shopping, Boardingpass, Map, information

The project started off with some paper prototypes of the check in after researching the user requirement. We made a User Journey and also looked at some value proposition / Customer Profile.

Gain Creators

Easy access to summoning a companion, Skip the queue for information, User-friendly features to reduce check-in time

Products & Services

Help booking / companion, Shopping, Practical information

Relievers

Practical complications, Map of elevators, Practical information / FAQ, Summon companion, Pre-order items

Gains

Availability, Summon companion, Comfort, Time saving, Less stress, Clear features

Pains

Hard to shop, Transport of own wheelchair, Legroom on the plane, Stressful experience, Practical complications, Stairs / Elevator, Transit to the plane, Security Control, Companion availability

Customer Jobs

Shopping A more ordinary airport experience Self-Check-In Save time at check-in Summon companion Avoid queues in information
