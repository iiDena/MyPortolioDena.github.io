Flower Webshop

Project Description

It may well be that the user says one, but the data says something else. Therefore, you must be able to make decisions based on data based on the best UX. You are going to work with A/B testing for the following case, where you have to set up hypotheses (experiments), collect and analyze the data and see if your hypotheses are correct or not wrong. So it is your data that drives the UX development for your case.

Details

  • Topic: Data Driven Design - A/B Testing
  • Softwares/tools: Visual Studio, HTML, Php, css, Javascript, Hotjar
  • Project Duration: 5 days

User Requirements

  • - The user must be able to buy flowers online and therefore it is important that the purchase form is user-friendly and clear.
  • - You should be able to read about the products and put them in a basket. (Shall not work - just measure how many do)
  • - You should be able to find information about the shop owner.
  • - You should be able to send flowers to each other with small extras such as wine or chocolate.

Project Requirements

  • - You have to set up different hypotheses that you have to test. So make different ones experiments.
  • - The prototype must be coded in HTML/CSS/JS/PHP
  • - Use the code templates that have been created delivered.
  • - The prototype must consist of 4 HTML pages: Front page, Product page, About page, Purchase form

Data & Hypotheses

We coded 2 similar webshops with some few changes (hypotheses). The Websites were coded in Visual Studio and both websites were under the same map. So when uploaded to Hotjar it would be random which page the test people would get into when clicking the link.

The point with our 2 hypotheses was to see if people would use the buttons we added more than the other buttons if they were on the website. Perhaps the customers would buy items faster if they details were more accessable and if the navigation was easier.

With our data we can conclude that the visitors of the page did in fact click on the buttons that we added more than the other navigations on the page, which made it easier for them to buy their products.

We collected this data looking at screen recordings,heatmaps and time on the website. The users on page B with more details was spent more minutes on.
